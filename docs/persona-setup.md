# 人格设定指南(Persona Setup)

Status: 配套 `templates/context/` 模板使用
Last updated: 2026-07-20

## 人格在系统里是怎么生效的

三个身份文件是她"是谁"的唯一来源,存放在 `context/`(gitignore 的个人内容,
仓库只提供模板):

| 文件 | 内容 | 进入哪些提示词 |
|---|---|---|
| `context/who_is_companion.txt` | 她是谁:名字、性格、说话方式、边界 | 每次唤醒 + 每条网页/Signal 聊天回复 |
| `context/who_is_human.txt` | 你是谁:称呼、偏好、雷区、约定 | 同上 |
| `context/now.txt` | 当下情况:最近的事、能力边界 | 同上 |

`companion_core/context.py` 在每次唤醒(`lifecycle.py`)和每个对话轮
(`dialogue.py`)开始时读取这三个文件,原文放进提示词的
`=== WHO YOU ARE ===`、`=== WHO YOUR HUMAN IS ===`、`=== CURRENT CONTEXT ===`
段。改完文件即刻生效,不需要重启任何服务。

## 为什么她现在"AI 味重"

身份文件还没写。提示词里她只知道自己"是个伴侣",没有名字、没有性格、
没有说话习惯,模型就会退回到默认的助手腔。系统层面能兜住的
(M8 对话人性化回归已经禁止了报告腔、主动汇报里程碑等),
但"像一个具体的人"只能靠身份文件给。

## 设置步骤

```bash
mkdir -p context
cp templates/context/who_is_companion.template.txt context/who_is_companion.txt
cp templates/context/who_is_human.template.txt context/who_is_human.txt
cp templates/context/now.template.txt context/now.txt
# 然后逐个填写 [ ] 里的内容,删掉不适用的示例行和注释行
```

## 写好人格文件的原则

1. **名字必须有。** 没有名字就没有"她"。
2. **性格用行为写,不用形容词堆。** "说话偏短,想清楚了才说"比"温柔体贴"
   有效十倍——模型能模仿行为,模仿不了标签。
3. **负面清单和正面描述一样重要。** 模板里的"反 AI 腔清单"
   (不说"作为AI"、不用客服结尾、不写要点报告、不复述提问)
   直接对着最常见的 AI 腔来源。
4. **不要写谎言。** 不给她"会泡茶"这类假身体设定;她真实的生活
   (定时醒来、写日志、有自己的网页窗口)已经足够构成一个真实的存在。
   编造的设定会和锚定门(grounding gate)冲突。
5. **`now.txt` 保持新鲜。** 它是三个文件里唯一需要经常改的;
   过时的"当下"会让她显得脱节。
6. **少即是多。** 三个文件加起来超过一百行,反而稀释性格。

## 与既有系统的配合

- **说话风格约束已在提示词里**:简体中文、聊天不写报告、不主动谈系统实现
  (`dialogue.py` 的边界段 + M8.5 人性化回归测试锁定)。
- **记忆会自动强化人格一致性**:M8 管家接受的事实与偏好、M12 语义检索
  召回的相关记忆,都会持续进入提示词,她会越聊越"记得你们是谁"。
- **人格文件不进 git**:属于 `.gitignore` 的个人内容;换机器/迁移 Pi 时
  按 M6 迁移清单手动带走(`context/` 在保留清单里)。

## 快速自检

设置完发一句"最近怎么样"到 `/chat`,如果回复:

- 有名字感、像短消息、不带要点符号 → 生效了
- 仍然"作为你的伴侣,我随时为你服务" → 检查 `context/` 文件是否
  真的存在且非模板占位状态(`[ ]` 是否都填了)
